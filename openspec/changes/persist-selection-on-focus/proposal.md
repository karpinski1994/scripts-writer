# Proposal: Persist Selection on Focus

## Summary

Add a visual persistence layer to keep the blue selection highlight visible when the user clicks on the BubbleMenu input/tooltip, ensuring users can always see exactly which text area will be regenerated by AI.

## Problem

When users select text in the Script Editor and the BubbleMenu appears, clicking on the input field to type an instruction causes the selection highlight to disappear. This happens because:

1. The selection is stored in React state (`storedSelection`, `storedSelectionPos`)
2. When focus moves to the input field, the editor loses focus
3. Prosemirror clears the selection render
4. The user can no longer visually see which text will be rewritten

Currently the code attempts to restore selection via `handleBubbleMenuFocus` on `onFocus`, but the blue background highlight that makes the selection visually distinct is lost when the editor loses focus.

## Proposed Solution

Use a custom Tiptap extension with Prosemirror Decorations to persist a visual selection highlight that survives focus loss. The extension will:

1. Track the stored selection position (`storedSelectionPos`) 
2. Apply a decorative CSS class (`.persistent-selection`) to that range
3. This decoration persists even when the editor loses focus to the BubbleMenu input
4. Ensure the CSS applies a blue background that survives focus change

## Requirements

- **REQ-1**: Selected text must show a blue background highlight that persists when clicking the BubbleMenu input
- **REQ-2**: The highlight should use the same blue color as the native selection for visual consistency
- **REQ-3**: The highlight should auto-clear when the user clicks away or after rewrite completes
- **REQ-4**: No modifications to the document content — purely visual decoration

## Non-Goals

- This is NOT about keeping the text selection active/functional for editing
- This is NOT about modifying Tiptap's core selection behavior
- This is NOT about adding new storage mechanisms

## Risks

- **Risk**: Tiptap v3 has different module exports than v2 — may need `require()` for dynamic imports
- **Risk**: Bubbling focus events may interfere with other editor behaviors
- **Mitigation**: Use a lightweight decoration approach that doesn't block other plugins
